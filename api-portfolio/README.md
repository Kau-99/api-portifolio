#  API de Gerenciamento de Livros

Bem-vindo(a) ao meu projeto! Desenvolvi esta API RESTful como parte do meu portfólio para consolidar e demonstrar meus conhecimentos na construção de aplicações Back-end robustas, seguras e escaláveis utilizando Node.js.

##  Sobre o Projeto

O objetivo deste projeto é fornecer um sistema completo de CRUD (Create, Read, Update, Delete) para o gerenciamento de um catálogo de livros. Em vez de criar apenas uma API básica, decidi elevar o nível técnico aplicando padrões de mercado.

###  Principais Diferenciais (Features)
* **Arquitetura MVC:** Separação clara de responsabilidades entre Rotas (`routes`) e Lógica de Negócios (`controllers`), facilitando a manutenção e escalabilidade do código.
* **Banco de Dados Real:** Integração com **SQLite** para persistência de dados real, utilizando consultas assíncronas (`async/await`) e SQL nativo.
* **Segurança Avançada:** * Implementação do **Helmet** para proteger os cabeçalhos HTTP contra vulnerabilidades web conhecidas.
  * Implementação do **Express Rate Limit** para prevenir ataques de força bruta e DDoS (limite de 100 requisições por IP a cada 15 minutos).
* **Tratamento de Erros e Validação:** Blocos `try/catch` em todas as operações de banco de dados para evitar crashes (quedas do servidor) e validação rigorosa dos tipos de dados enviados nas requisições (`bad requests`).

##  Tecnologias Utilizadas

* **Node.js** e **Express** (Criação do servidor e roteamento)
* **SQLite** e **sqlite3** (Banco de dados relacional e motor SQL)
* **Helmet** (Segurança de cabeçalhos HTTP)
* **Express Rate Limit** (Proteção contra sobrecarga de requisições)
* **Cors** (Controle de acesso a recursos de origem cruzada)
* **Nodemon** (Ambiente de desenvolvimento)
